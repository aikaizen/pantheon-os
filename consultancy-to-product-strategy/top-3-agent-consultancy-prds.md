# Top 3 Agent Consultancy PRDs

## Purpose
This document ranks the top three product bets for a consultancy-focused AI/agent venture and explains why each deserves attention.

## Evaluation criteria
Each concept is scored on:
- pain intensity
- speed to first revenue
- repeatability across clients
- long-term defensibility
- implementation complexity
- ability to become a real software business

## The top 3
### 1. Institutional Memory Engine
Why it ranks first:
- every consultancy loses value through knowledge fragmentation
- the pain is immediate and easy to explain
- it improves both service delivery and future product leverage
- it becomes the shared foundation for other agent products

What it solves:
- rediscovery of prior work
- weak handoffs
- low reuse of templates, decisions, and lessons learned
- inconsistent context for humans and agents

Why it matters strategically:
- it creates compounding leverage across all future work
- it is useful internally first, then sellable externally

### 2. Long-Horizon Workflow Agent
Why it ranks second:
- long-running workflows are expensive and failure-prone
- buyers already understand the cost of missed follow-up and project drift
- it has strong recurring value when tied to onboarding, delivery, audits, or implementation programs

What it solves:
- weak coordination over multi-week work
- invisible blockers
- no clear next action
- poor continuity across owners and milestones

Why it matters strategically:
- it is a strong product wedge for services-heavy teams
- it can expand into a broader operating layer over time

### 3. Agent Runtime Control Plane
Why it ranks third:
- it becomes necessary once agents are operating at scale
- it is high value, but the pain usually appears after the first two products are in active use
- it is more infrastructure-heavy and less urgent as an initial wedge

What it solves:
- unclear runtime state
- weak approvals and governance
- poor cost visibility
- unsafe multi-client agent operations

Why it matters strategically:
- it is the control and trust layer that supports scaled adoption
- it becomes more valuable as the consultancy evolves into a multi-client product operator

## Priority recommendation
Recommended build order:
1. Institutional Memory Engine
2. Long-Horizon Workflow Agent
3. Agent Runtime Control Plane

## Why this order works
### Memory first
Memory creates immediate internal leverage and improves delivery quality across every engagement.

### Workflow second
Workflow turns that memory into operational follow-through and an externally sellable coordination product.

### Control plane third
The control plane becomes more important once multiple agents, workflows, and clients are active at the same time.

## Strategic interpretation
If the company wants:
- fastest leverage: build memory first
- strongest workflow wedge: build workflow second
- strongest infrastructure moat: build control plane after real runtime complexity exists

## Final recommendation
For a consultancy-to-product transition, the most rational sequence is:
- use the Institutional Memory Engine to capture and reuse knowledge
- use the Long-Horizon Workflow Agent to operationalize that knowledge over time
- use the Agent Runtime Control Plane to govern the growing system safely

These three products fit together as a stack rather than as disconnected bets.
