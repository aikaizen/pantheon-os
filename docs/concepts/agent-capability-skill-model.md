# Agent, Sub-Agent, Capability Set, and Skill Model

## Agent
An agent is a durable operating identity with mandate, scope, tools, memory, and review boundaries.

## Sub-agent
A sub-agent is an ephemeral execution entity created for a bounded task.

It should be:
- short-lived
- scoped
- replaceable
- reviewable

## Capability set
A capability set is the class of abilities an agent can bring to work.

Examples:
- marketing
- coding
- sales
- research
- operations
- synthesis

## Skill
A skill is a reusable operational procedure aligned to a requirement class.

Skills should become modular and composable so that agents can be assembled or extended according to domain needs.

## Model
- durable agent = identity + mandate
- ephemeral sub-agent = scoped execution
- capability set = what classes of work can be done
- skill = reusable procedure inside a capability set
