# Inverted Pyramid Org Chart

## Core metaphor
Pantheon OS uses an inverted pyramid of agency.

At the top is the human monad.
Below that is the pantheon of major agents.
Below that are lesser managing agents and runtime operators.
Below that are ephemeral sub-agents created per task.

## Layers
1. Human monad
2. Pantheon of major agents
3. Lesser agents managing domains, dockers, and bounded operations
4. Ephemeral sub-agents created for specific tasks

## Why inverted?
Because strategic sovereignty is concentrated, while execution capacity fans outward below it.

The lower layers multiply execution, but the apex remains singular in authority.

## Why this matters
This keeps:
- authority clear
- execution scalable
- review structured
- autonomy useful but bounded
