# Pantheon OS System Architecture

## Overview

Pantheon OS has four major surfaces:

1. Principles
2. Company Dashboard
3. Operating Terminal
4. Runtime / Portal

These are not four unrelated documents. They are four aspects of one operating system.

## Principles
The constitutional layer defines what the company is, what an agent is, how authority works, how power is bounded, and how escalation is handled.

## Company Dashboard
The dashboard is the strategic surface. It makes the company legible across ventures, operators, budgets, approvals, goals, and activity.

## Operating Terminal
The operating terminal is the tactical surface. It makes the pantheon steerable in real time across chats, terminals, panes, tools, logs, and summaries.

## Runtime / Portal
The runtime layer is the substrate beneath the visible surfaces. It includes dual harnesses, portal state, supervision, telemetry, and memory.

## PromptEngines relationship
Pantheon OS is a general system.
PromptEngines is the first internal deployment.
