# Workstream A — PromptEngines Dashboard MVP

## Objective
Ship the fastest credible PromptEngines-first dashboard inside Pantheon OS.

## Product stance
This is not a generic company dashboard.
It is the strategic control surface for the first live Pantheon OS deployment: PromptEngines.

## Required views
- Deployments: all companies on one page
- Company: deep dive on one company
- Agents: active PromptEngines agents only
- Operating model: constitution, portal logic, and terminal linkage

## PromptEngines requirements
- Reflect the real public PromptEngines surface area visible on promptengines.com
- Follow the PromptEngines / Lab Notes design language
- Avoid rounded bubble SaaS styling
- Show the metric families already being built for the dashboard:
  - Growth
  - Finance
  - Engagement
  - Reliability
  - Unit economics
  - User profitability
- Separate public products, prototypes, and internal Pantheon surfaces

## Agent-view requirements
Do not fabricate agents.
The PromptEngines agent tab should show the actual active roster for the company deployment and include:
- sitrep
- current work
- next work
- blockers
- direct message handoff into the terminal-manager lane

## Near-term outputs
- PromptEngines-first IA refresh
- sharper Lab Notes–referenced UI direction
- portfolio registry aligned with the public site
- metric-family framing across company and app views
- active-agent operations tab with handoff queue
- operating-terminal wireframe link from the dashboard

## Success condition
A PromptEngines dashboard exists that makes the company legible, respects Pantheon OS philosophy, and looks aligned with PromptEngines taste rather than generic dashboard-kit aesthetics.
