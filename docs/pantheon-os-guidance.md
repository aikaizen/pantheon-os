
# Pantheon OS Guidance Layer

## What was integrated from the Anthropic skill guide

- skills are first-class operational assets, not just prompt helpers
- every skill starts from concrete use cases, not vague abstractions
- progressive disclosure is standard:
  - frontmatter for triggering
  - `SKILL.md` body for execution
  - linked references / scripts / assets only when needed
- skills should be composable across agents
- skills sit above tools and connectors as the recipe layer
- important skills are tested for:
  - triggering
  - functional success
  - performance / workflow improvement
- Prompt Engines skill organization should include:
  - creation skills
  - workflow automation skills
  - connector-enhancement skills
  - company-os skills
  - venture-specific skills
- skill governance should track:
  - owner
  - version
  - status
  - review cadence
  - venture scope

## Practical meaning for Prompt Engines

- all major agents should use the same company skill standard
- the pantheon is not reduced by skills; skills sharpen execution
- constitution aligns agents
- skills operationalize repeatable work
- memory preserves what is learned
- connectors provide access

## Paperclip evaluation

Short version:
- strong reference system
- not the right core platform for Prompt Engines v1

Why it is interesting:
- high conceptual overlap with multi-agent company orchestration
- strong thinking on org charts, governance, budgets, heartbeats, and goal alignment
- supports multiple agent runtimes
- already treats skills seriously

Why it is not the base:
- too heavy for current Prompt Engines v1
- more opinionated toward a zero-human-company framing
- not obviously constitution-first or archetype-first
- more rigid company/task-manager ontology than the pantheon + manifests model
- risks forcing the company model to fit the tool

Recommended stance:
- reference value: high
- direct v1 fit: medium-low
- adoption recommendation: borrow patterns, do not adopt as core substrate
