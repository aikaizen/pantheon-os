Onboarding

Connect Existing Setup

This is the primary adoption path. PantheonOS should attach to the operator's existing agent estate — computers, dockers, models, Telegram bots/channels, controller workflows — before it asks them to rebuild anything.

Default posture

Observe first

Start in observe mode. Prove PantheonOS can see the existing setup before it is allowed to message or control it.

Adopt, do not replace

Existing agents keep their identity, soul, memory, and channels. PantheonOS becomes the shared surface above them.

Required steps

  • register deployment metadata
  • register existing runtimes
  • register existing controller/overseer agents
  • register existing channels
  • bind persona agents, runtimes, controllers, and channels
  • run the state engine and verify the topology is legible
  • upgrade bindings from observe to message or control only when safe

Minimum viable attachment

PantheonOS becomes useful as soon as it can show one principal lane, one persona lane, one runtime, one controller relationship, and basic health / heartbeat / summary state on a single surface.
