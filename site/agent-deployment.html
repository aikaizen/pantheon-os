Definitional doc

Agent Deployment Layer

PantheonOS is the operating surface. The runtime underneath may be Hermes, OpenClaw, NanoClaw, or PantheonOS itself. In v1, PantheonOS should white-label Hermes as the default packaged deployment path while still supporting attachment of existing orchestrators.

Core distinction

PantheonOS

The shared observation and intervention surface. This is the product the operator experiences.

Runtime system

The underlying Dockerized engine that actually hosts and executes the agents: Hermes, OpenClaw, NanoClaw, or later a Pantheon-native runtime.

Two deployment paths

1. Connect Existing Runtime

Register an existing Dockerized Hermes, OpenClaw, NanoClaw, or Pantheon runtime inside PantheonOS without rebuilding the agent estate from scratch.

2. Deploy with Pantheon

Provision a Pantheon-branded deployment package. In v1, this should usually mean a white-labeled Hermes runtime with Pantheon topology and onboarding conventions layered on top.

White-labeled Hermes

“Deploy with Pantheon” should not force a Pantheon-native runtime before that runtime exists. Instead, PantheonOS should package Hermes as the default runtime substrate while making the relationship explicit in registry and state output.

support_levelFirst-party, adapter-supported, or experimental

Operator truth

The operator should always be able to answer:

  • what runtime system is this using?
  • is it attached or provisioned?
  • is it white-labeled Hermes or a native runtime?
  • which controller maintains it?
  • which persona agents and channels are bound to it?

Registry implication

Runtimes must store runtime identity, branding mode, deployment mode, support level, image, version, and bindings. Deployments must declare their default runtime path and supported runtime systems.

PromptEngines v1 default

For PromptEngines, the default packaged path should be:

  • PantheonOS product experience
  • white-labeled Hermes runtime
  • Docker packaging
  • controller-based maintenance
  • observe-first onboarding
