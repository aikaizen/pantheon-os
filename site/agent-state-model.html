◆Overview

Pantheon OS tracks agent state semantically. Each state represents a distinct operational mode — not just activity, but the nature of that activity.

◆Required States

idle
Operationally idle. Not merely silent — actively waiting for work.
thinking
Processing, reasoning, planning next action.
executing
Running code, calling tools, performing work.
waiting_on_tool
Blocked on external tool response.
waiting_on_human
Needs human input, approval, or decision.
blocked
Cannot proceed — dependency, error, or constraint.
reviewing
Examining output, checking quality, verifying.
error
Encountered an error requiring intervention.
complete
Task finished successfully.
paused
Suspended by operator intervention.
Idle must mean operationally idle, not merely silent. An agent that is connected but not working is idle. An agent that has disconnected is not idle — it is in an unknown state.
