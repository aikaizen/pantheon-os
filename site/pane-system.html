◆Overview

Panes are the atomic working surfaces of the Operating Terminal. Each pane has a type, a data source, and semantic identity within a workspace.

◆Core Pane Types

Channel Pane
Message threads from Telegram, WhatsApp, and internal channels
Execution Pane
Live shell, tmux, and docker console output
Summary Pane
Agent state, blockers, and current task overview
Artifact Pane
Files, outputs, and deliverables from agent work
Portal Pane
Task state, checkpoints, and handoff notes
Telemetry Pane
Metrics, heartbeat status, and resource usage
Oversight Pane
Harness state, control actions, and approval queue

◆Pane Principles

  • Panes should be composable — combine freely in workspace layouts
  • Panes should be saveable — persist workspace configurations
  • Panes should have semantic identity — not just windows, but typed surfaces
  • Panes should support drill-down — from summary to raw execution
