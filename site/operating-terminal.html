Operating Terminal

← Pantheon OS

◆Purpose

The Operating Terminal is the tactical command-center surface of Pantheon OS.

It exists to help a human operator work across sovereign persona agents, controller agents, and runtime health on one screen.

◆Core Idea

This is not just a terminal emulator. It is a TMUX-like multi-pane operating surface that can show:

  • Persona chats — Telegram, WhatsApp, CLI, internal threads
  • Controller terminals — host-side overseer sessions such as Claude Code or Codex
  • Runtime health — docker status, model config, process state, adapter reachability
  • Logs — runtime output and debug streams
  • Summaries — state, blockers, handoffs, venture context
  • Artifacts — files, outputs, deliverables
  • Interventions — observe, message, control actions

◆Required Capabilities

  • Multi-pane layout with saved workspaces
  • Persona channel panes for Telegram, WhatsApp, and CLI lanes
  • Controller terminal panes for host-side runtime management
  • Runtime health panes for docker/model/process telemetry
  • Artifact and summary panes for deliverables, blockers, and sitreps
  • Permission-aware interventions: observe → message → control

◆Primary Pane Types

  • Persona Channel Pane — conversation with a sovereign persona agent
  • Controller Terminal Pane — host-side overseer console
  • Runtime Health Pane — runtime, model, and adapter status
  • Summary Pane — current state, blockers, and next actions
  • Artifact Pane — files, outputs, links, approvals
  • Intervention Pane — observe/message/control actions and audit history

◆Core Controls

  • Open / focus a persona lane
  • Attach to a controller terminal
  • Inspect runtime health and logs
  • Request summary / escalate / reassign
  • Pause / resume / restart when the binding allows it
