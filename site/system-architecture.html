System Architecture

← Pantheon OS
Surface 1

Principles

The constitutional layer defines what the company is, what an agent is, how authority works, how power is bounded, and how escalation is handled. Source of truth lives in constitution/ and registry/.

Surface 2

Company Dashboard

The dashboard is the strategic surface. It makes the company legible across ventures, operators, budgets, approvals, goals, and activity. The current MVP is the seed, not the full product.

Surface 3

Operating Terminal

The operating terminal is the tactical surface. It makes the pantheon steerable in real time across chats, terminals, panes, tools, logs, and summaries. It is the command-center IDE for supervising and steering teams of agents.

Surface 4

Runtime / Portal

The runtime layer is the substrate beneath the visible surfaces. The canonical deployment model is: one host machine with an overseer agent, one Hermes instance per Docker container, and up to four agent slots inside each Hermes runtime. Portal state, supervision, telemetry, event flow, memory, and agent state transitions all sit here.

Integration: These are not four unrelated documents. They are four aspects of one operating system. The Principles layer governs the Dashboard. The Dashboard surfaces what the Terminal needs to steer. The Terminal operates through the Runtime. The Runtime preserves state via the Portal.
PromptEngines relationship: Pantheon OS is a general system. PromptEngines is the first internal deployment.
