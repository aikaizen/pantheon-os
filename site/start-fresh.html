Onboarding

Start Fresh

Use this path when the operator is new to agents and does not already have a runtime/controller/channel topology. The goal is to create the smallest useful PantheonOS deployment without changing the underlying model.

Minimum useful topology

  • 1 principal
  • 1 persona agent
  • 1 runtime
  • 1 controller
  • 2 channels: principal CLI + persona channel

Required steps

  • define persona agents
  • define at least one runtime
  • define at least one controller
  • define principal and persona channels
  • bind the topology explicitly
  • start in observe mode where possible, then enable message, then control
  • run the state engine and verify the deployment renders correctly
